## 管理のベストプラクティス

1. **依存関係は明示的に記載**
   - Issue本文またはコメントに記載
   - プロジェクトボードのカスタムフィールド活用

2. **ラベルで状態を可視化**
   - `blocked`: 依存Issueの完了待ち
   - `ready`: 依存関係解消、作業可能

3. **定期的な依存関係レビュー**
   - 週次でブロッカーを確認
   - 不要な依存関係は削除

4. **並行作業の最大化**
   - 独立したタスクは同時進行
   - チーム開発時は依存関係を考慮した割り振り
